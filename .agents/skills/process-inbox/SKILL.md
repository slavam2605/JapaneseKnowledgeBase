# Разбор inbox.md

Пользователь накидывает в `japaneseCards/data/inbox.md` вперемешку слова, кандзи,
грамматику, языковые вопросы и иногда мета-инструкции. Этот скилл — **триаж**: разобрать поток,
разложить материал по бэклогам и ответить на вопросы. **Карточки тут НЕ создаются** — это следующий
шаг, скилл `japanese-cards` (читает те же бэклоги).

Все пути — относительно `japaneseCards/`. Общие соглашения (безопасность Anki, синтаксис ввода,
словарь) — в `japaneseCards/knowledge/conventions.md`.

## Целевые файлы

| Куда | Что туда идёт |
|---|---|
| `data/words.md` | обычная лексика для колоды `新日本語::Слова` |
| `data/kanji.md` | всё «на чтение»: кандзи-чтения, **числа, даты, счётчики, месяцы, дни недели, время** → `新日本語::Чтение кандзи` |
| `data/grammar.md` | грамматические конструкции для `新日本語::Грамматика` |
| `data/answers.md` | ответы на языковые вопросы (синонимы, узус, чтения, грамматика) |

Разобранные строки **удаляются из `inbox.md`** (так договорено в шапке inbox.md и README).
Заголовок и преамбулу inbox.md (строки до `---`) не трогать.

## Процедура

1. Прочитать `data/inbox.md` целиком. Каждая непустая строка ниже разделителя `---` — одна запись.
2. Классифицировать каждую запись (см. «Классификация»).
3. **Спросить про неоднозначное ДО раскладки.** Если есть «кандидаты в грамматику» (записи без явной
   пометки, но хорошо ложащиеся в паттерн-колоду) или другие записи, где нужно решение пользователя —
   **не раскладывать их молча и не класть как заглушку в `words.md`**. Вынести их списком прямо в
   диалог (одна строка — один кандидат + короткая причина) и дождаться ответа. Однозначные записи
   можно разложить сразу, не дожидаясь ответа по кандидатам.
4. Внести записи в нужный файл по итогам ответа; на вопросы написать ответ в `answers.md`.
5. **Не выдумывать чтения/значения.** Для заполнения `kanji.md` (нужны Kana + Meaning) и для
   фактической части ответов использовать локальный словарь (детали — в `conventions.md`):
   ```
   python3 japaneseCards/tools/dict_lookup.py 銀行 切符 たぶん
   python3 japaneseCards/tools/dict_lookup.py --kanji 急行
   ```
6. Удалить разобранные строки из `inbox.md`. То, что не разобрано (см. «Мета-инструкции») или ждёт
   ответа пользователя по кандидатам — оставить в inbox с пометкой и проговорить пользователю.
7. В конце дать краткую сводку: сколько ушло в words / kanji / grammar, сколько вопросов отвечено,
   что осталось висеть и почему.

## Классификация

Сначала смотри на **маркеры намерения** (синтаксис — в `conventions.md` разд. «Синтаксис ввода»):

- Комментарий после `|` или пробела — свободная инструкция на русском. Сохранять её рядом со словом
  при переносе в бэклог (она пригодится при создании карточки).
- Вопросительные слова в комментарии («?», «как читается», «нормально говорить?», «насколько…»,
  «vs», «почему») → это **вопрос**.
- `>кандзи` / `>слова` — принудительный выбор бэклога.

Дальше по сути записи:

- **Вопрос** (есть вопросительная часть) → ответить в `answers.md`. Если запись это слово + вопрос
  (напр. `デパート насколько это современно?`), ответить на вопрос **и** по умолчанию завести слово
  в `words.md`; если из ответа следует, что слово архаично/не нужно — слово не заводить, отметить
  это в ответе.
- **«На чтение»** — числа, время (`７時`, `午前0時10分`), счётчики (`~番線`), даты, месяцы, дни недели,
  дни месяца → `kanji.md`. Сюда же отдельные кандзи-слова, у которых цель «увидеть кандзи → прочитать».
- **Грамматика / паттерны** — конструкция/частица/суффикс с пояснением «грамматика …»
  (`へ из 5 урока`, `грамматика で`, «после чего ставим に») → `grammar.md`. Колода `Грамматика` —
  это паттерн-тип (cloze + рандомизация): кроме конструкций сюда же логически ложатся «какая
  **частица** при слове» (`友達に会う`) и «какой продуктивный **суффикс** передаёт смысл» (`〜すぎる`,
  `〜やすい/にくい`). Полный scope — в `conventions.md` (таблица колод) и `card-types.md` §3.
  - **Дефолт — консервативный.** В `grammar.md` переноси по `grammar.md` **только при явной пометке**
    (`>грамматика`/«грамматика …»/прямое указание пользователя). Пользователь сам помечает
    большинство случаев.
  - **Но спрашивай — до раскладки.** Если запись без пометки **явно** хорошо ложится в паттерн-тип
    (типичный кейс — частица при глаголе или продуктивный суффикс), **не переноси молча и не клади
    заглушкой в `words.md`**: вынеси её прямо в диалог в список «кандидаты в грамматику» (одна строка —
    один кандидат + короткая причина) и дождись ответа пользователя, куда класть. Раскладывай такой
    кандидат только после ответа. Сомневаешься, паттерн ли это — не предлагай, клади по дефолту.
- **Лексика** — всё остальное (слова, фразы, выражения) → `words.md`.

Пограничный случай «слово или чтение?»: если запись дана **каной** и это обычная лексика — `words.md`;
если суть в том, чтобы прочитать **кандзи/число** — `kanji.md`. При сомнении клади в `words.md`
и отметь это в сводке.

### Мета-инструкции (не материал, а указание)

Часть строк — не запись, а инструкция. Два подвида:

- **«Разверни список»** — `Все дни недели`, `12 месяцев в колоду произношений`, `Все дни — тсуитачи,
  тоока… от 1 до 31`, `Все слова из уроков…`. Развернуть в конкретные записи и разложить по правилам
  (дни недели / месяцы / дни месяца / числа → `kanji.md`, проверив чтения через `dict_lookup.py`).
  Развёрнутое — внести, исходную мета-строку удалить.
- **Действие-задача** — `Все добавленные найти с клодом в старых колодах и удалить`. Это не материал
  для бэклога, а отдельная операция (и часто запись в **чужие** карточки → действуют правила
  безопасности из `conventions.md`: чужое не трогать без явного согласия). Такие строки
  **не удалять молча**: вынести их пользователю как отдельный список «требует действия» и спросить,
  делать ли. Даже после согласия карточки не удалять: пометить `to_remove` и заморозить все их
  стороны. Оставить инструкцию в inbox, пока не решено.

## Форматы записи в бэклогах

- **`words.md`** — одна запись на строку, сохраняя комментарий через `|`. Примеры:
  ```
  銀行
  急行 | vs 特急, добавить "не"-синонимы в HintRU
  あの方はどなたですか | вежливая фраза
  ```
  Если файл пуст — добавить короткий заголовок-преамбулу в стиле соседних файлов.
- **`kanji.md`** — формат из шапки файла: `Kanji | Kana | Meaning` (опц. `| Hint`). Kana и Meaning
  брать из `dict_lookup.py`, не из памяти. Пример: `急行 | きゅうこう | скорый поезд`.
- **`grammar.md`** — добавлять пункт в раздел `## В очереди` (убрать `_(пусто)_`, если он там).
  Одна строка — одна конструкция с кратким описанием задачи, напр.:
  ```
  - へ (направление) — почему へ, а не に; урок 5
  - で (средство/место действия) — 私は電車で会社へ行きます
  ```

## Формат ответов в answers.md

Каждый ответ начинать с **цитаты вопроса** (а не с номера строки: разобранные строки удаляются из
inbox, нумерация съезжает). Например:

```
## デパート — насколько современно? Норм использовать?
Слово живое и нейтральное, ... . Завёл в words.md.
```

Ответы — **по-русски, кратко и по делу**, уровень аудитории N5–N4 (см. `conventions.md`).
Языковые факты (чтения, части речи, значения) сверять через `dict_lookup.py`, а не сочинять.
В вопрос не превращать в карточку без явной просьбы.

## После разбора

- Сводка пользователю (сколько куда, что осталось «требует действия»).
- Напомнить, что создание карточек из наполненных бэклогов — это скилл `japanese-cards`.
