# Создание японских карточек

Этот скилл описывает **что делать** при наполнении карточек (какой набор данных взять, в каком
порядке, что сообщить пользователю). **Как именно** заполнять поля и верстать карточку — НЕ здесь,
а в общедоступных knowledge-файлах (соблюдать строго, не дублировать):

- **Общее для всех типов** — `japaneseCards/knowledge/conventions.md`: безопасность Anki, имена
  колод/типов, теги, suspend/no-confirm, синтаксис ввода (`| ? >кандзи`), фуригана, локальный
  словарь, **механика записи `anki_add.py`**.
- **Как создать карточку конкретного типа** (поля, правила каждого поля, алгоритмы `Hint`/`Meaning`,
  шаблон, чек-лист) — per-type файл, **читать перед созданием**:
  - слова → `japaneseCards/knowledge/words-cards.md`
  - чтения кандзи → `japaneseCards/knowledge/kanji-cards.md`
  - грамматика/паттерны → `japaneseCards/knowledge/grammar-cards.md`
- Логика «что каким типом учить» и границы — `japaneseCards/knowledge/card-types.md`.

## Входные данные (какой набор брать)

Список приходит как аргумент скилла. Поддерживаемые формы:
- **Инлайн-список** — слова через запятую/с новой строки: `切符、たぶん、銀行員`.
- **Ссылка на файл/диапазон** — напр. «из data/words.md строки 1–20»: прочитать файл самому и взять
  указанные записи.
- **Каталог MNN** — напр. «сделай карточки из reference/mnn/lesson05.md»: прочитать таблицу
  `かな｜漢字｜Перевод｜Пометка` (формат — `japaneseCards/knowledge/mnn-catalog.md`) и взять слова
  оттуда. Это источник, не очередь: каталог не меняем, что залито — фиксируем в `STATUS.md`.
- **Пусто (дефолт) — взять ВСЁ из трёх бэклогов разом.** Если скиллу явно не дали другой набор
  данных, обработать **все** записи из `data/words.md`, `data/grammar.md` и `data/kanji.md`, каждый
  файл — в свою колоду (`新日本語::Слова`, `新日本語::Грамматика`, `新日本語::Чтение кандзи`). Не
  спрашивать, какой список брать, и не ограничиваться одним блоком — это полный прогон всех трёх
  файлов. Файлы могут быть пустыми — тогда по этому типу просто нечего создавать.

**Точка входа для нового материала — `data/inbox.md`:** пользователь накидывает туда слова / кандзи /
грамматику / вопросы вперемешку. Раскладка по целевым файлам и ответы на вопросы — это **отдельный
шаг** (скилл `process-inbox`), ДО создания карточек. На языковые вопросы отвечать пользователю, в
карточки не превращать без явной просьбы.

По каждому слову пользователь даёт минимум (само слово; опционально чтение/значение/пометки).
Недостающие поля и интерпретация свободной инструкции после `|` — по правилам per-type файла
(синтаксис самого `|`-ввода — в conventions.md).

## Процедура

1. **Определить набор данных** (см. выше) и разнести записи по типам/колодам (слова / чтение / паттерн).
   Если запись неоднозначна по типу — решить по `card-types.md`; сомнительных кандидатов в паттерн-
   колоду не молча раскладывать, а показать пользователю.
2. **Для каждого типа открыть его per-type файл** и заполнить поля строго по нему (`words-cards.md` /
   `kanji-cards.md` / `grammar-cards.md`). Языковые факты (чтения/значения) — сверять по
   `dict_lookup.py` (conventions.md, «Локальный словарь»), не сочинять.
3. **Показать черновик** (таблица полей; для слов — колонка `jlpt/common`; видно, что свободная
   инструкция пользователя учтена в нужном поле).
4. **Перед массовым добавлением сообщить**, сколько и каких карточек из какого источника создаётся
   (правило безопасности №4 в conventions.md). Синхронного «подтверждаете?» не спрашивать.
5. **Грамматика — обязательный gate проверки субагентом ДО записи.** Тип «Грамматика» сложнее
   остальных, поэтому черновик грамматических карточек сначала проверяет отдельный read-only
   субагент-ревьюер (адверсариально, против всех правил), замечания исправляются, и только потом
   карточки идут в Anki. Полный порядок и список проверок — в `grammar-cards.md`, раздел
   «Обязательная проверка субагентом перед записью». Для слов/кандзи этот шаг не нужен.
6. **Создать через `anki_add.py`** (механика, теги, suspend, проверка рендера — в conventions.md,
   «Механика записи»). Готовить только данные карточек в `batch.json`.
7. **Проверить результат** через `cardsInfo` по чек-листу из per-type файла (число карточек,
   `is:suspended`, фуригана/маркеры/стороны рендерятся).
8. Пользователь сам отсматривает карточки в Anki и снимает `suspend` + `ai_generated::to_review`.
