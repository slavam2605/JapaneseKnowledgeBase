# Разбор дубликатов старое ↔ новое

Многие новые карточки `新日本語::*` дублируют слова из старого архива `0. Архив::日本語::*`.
Этот скилл находит такие пары через локальный эмбеддинг-индекс, **переносит картинку** из старой
карты в новую (у старых MNN-карт часто есть картинки, у новых — нет), помечает старую карту
`to_remove`, а новую — `ai_generated::deduped`, чтобы повторный прогон её не трогал.

**Границы безопасности (строго, см. `japaneseCards/knowledge/conventions.md`):**
- Новые карты `新日本語::*` — **наши** (`ai_generated`), правим свободно (вписываем картинку).
- Старые карты архива — **чужие**. Разрешено только повесить тег `to_remove` (пользователь прямо
  санкционировал этот маркер в дизайн-обсуждении). **Удалять старые карты нельзя** — их
  пользователь удаляет сам в Anki, просмотрев `tag:to_remove`. Никаких `deleteNotes` по архиву.

Предпосылка: эмбеддинг-индекс архива уже собран (см. `japaneseCards/tools/EMBEDDINGS.md`). Если
базы `data/anki_index.sqlite3` нет или архив в неё не выгружен — сначала собрать по EMBEDDINGS.md.

> **Архив НЕ статичен — освежать каждый прогон (шаг 1).** Пользователь между проходами удаляет в
> Anki архивные карты с `to_remove`. Удаление в Anki **не трогает** локальный SQLite: строка
> остаётся с `is_present=1`, и скан матчит новые карты против **призраков** (мёртвые `note_id`).
> Симптом — куча SURE-«дублей» против несуществующих архивных заметок (проверка через AnkiConnect
> вернёт `null` на их id). Лечится переэкспортом архива (шаг 1): он выставит `is_present=0` на
> удалённых, а скан их фильтрует. **Переэкспорт дёшев** (read-only AnkiConnect → локальный SQLite,
> секунды, в Anki ничего не пишет); пересчёт эмбеддингов инкрементальный и для удалений **не
> запускается** (векторы живых заметок сохраняются). Так что архив надо гнать через export каждый раз.

## Процесс

### 1. Освежить индекс — и новые колоды, и архив

Экспорт/сборка идут по **точной** колоде (`--deck`), `is_present`-свип ограничен рамкой этой
колоды — поэтому переэкспорт любой колоды безопасен и колоды друг друга не портят. Гнать **и
новые, и архивные** (архив — чтобы выпали удалённые пользователем `to_remove`-карты, см.
предупреждение выше):
```bash
cd japaneseCards
export HF_HUB_OFFLINE=1
for d in \
  '新日本語::Слова' '新日本語::Чтение кандзи' '新日本語::Грамматика' \
  '0. Архив::日本語::Kanji::Kanji' '0. Архив::日本語::Kanji::Kanji words' \
  '0. Архив::日本語::Words::Grammar' '0. Архив::日本語::Words::Words MNN1-2' \
  '0. Архив::日本語::Words::Words MNN3' '0. Архив::日本語::Words::Words 完全マスターN2'; do
  .venv/bin/python tools/export_anki_notes.py --deck "$d"
  .venv/bin/python tools/build_embeddings.py  --deck "$d"
done
```
(Экспорт идемпотентен, сборка инкрементальна — гонять безопасно. `Kanji writing` НЕ грузить: те же
заметки, что `Kanji`. Для чистых удалений архива `build_embeddings` — почти no-op. Детали —
EMBEDDINGS.md.)

### 2. Найти кандидатов (read-only скан)

```bash
.venv/bin/python tools/dedup_scan.py
```
Скрипт **ничего не пишет в Anki** — только генерирует:
- `data/removal_candidates.json` — все пары (машинное);
- `data/removal_candidates_review.md` — то же по тирам, для человека.

Механика: для каждой новой заметки берёт top-N ближайших по косинусу из архива **∪** все заметки
с совпадающим нормализованным чтением (reading-safety-net — дубль не теряется, даже если косинус
просел, напр. `おじいさん`/`お祖父さん` cos 0.78). Новые заметки с тегом `ai_generated::deduped`
пропускаются (идемпотентность); `--all` — пересканировать всё.

**Тиры (решать по ПОВЕРХНОСТНОЙ ФОРМЕ, не по смыслу):**

| Тир | Что значит | Действие по умолчанию |
|---|---|---|
| `SURE` | чтение И письменная форма совпали | **дубль** — обрабатывать (косинус вторичен) |
| `REVIEW` | чтение то же, запись разная, cos ≥0.80 | обычно орфо-вариант (`子供`/`子ども`, `20歳`/`二十歳`) — **дубль**, но глянуть |
| `HOMOPHONE` | чтение то же, запись разная, cos <0.80 | скорее РАЗНЫЕ слова (`例`/`零`, `着る`/`切る`) — **НЕ дубль**, проверить и обычно пропустить |
| `SEMANTIC` | РАЗНОЕ чтение, высокий косинус | обычно НЕ дубль (`小さな`/`小さい`) — оставить обе |

### 3. Отсудить пары

- `SURE` — принимать как дубли без лишних сомнений.
- `REVIEW` — просмотреть; орфографический вариант того же слова = дубль. Если реально разные слова
  (редко) — пропустить.
- `HOMOPHONE` / `SEMANTIC` — по умолчанию **НЕ** дубли: чтение совпало случайно или слова просто
  близки. Помечать дублем только если по форме и смыслу это очевидно то же слово.
- Одна новая карта может совпасть с несколькими старыми (напр. `さけ` → `酒` и `サケ`): выбрать
  действительно покрывающие; на каждую подтверждённую старую вешается `to_remove`.
- Реально неоднозначное — **не угадывать**: оставить в `removal_candidates_review.md` и спросить
  пользователя, эти строки не трогать тегами.

### 4. Применить решения (AnkiConnect, localhost:8765, проверить `version`)

На **каждую подтверждённую пару** (новая ← старая):

1. **Перенести картинку** (если у новой её ещё нет). `notesInfo` по обеим заметкам. Если поле
   `Image` новой карты **пусто**, а в любом поле старой есть `<img ...>` — скопировать этот тег
   `<img src="…">` **как есть** в поле `Image` новой карты (`updateNoteFields`).
   - Медиа-файл общий на всю коллекцию → **не** звать `storeMediaFile`, **не** трогать сам файл.
     После того как пользователь позже удалит старую заметку, файл останется referenced новой картой,
     и Anki его не подчистит.
   - Перенос **обязательно до** пометки `to_remove` (пока старая карта ещё существует).
   - Перенос картинки — это **изменение содержимого** → сразу после `updateNoteFields`
     **заморозить** карты новой заметки (`suspend`) и вернуть тег `ai_generated::to_review`
     (правило «любая правка → снова на ревью», см. conventions).
   - Если у новой уже есть картинка или у старой её нет — шаг пропустить.
2. **Пометить старую** заметку тегом `to_remove` (`addTags`). Не удалять.
3. **Пометить новую** заметку тегом `ai_generated::deduped` (`addTags`) — маркер «продедуплено».
   Сам по себе этот тег **НЕ** отправляет карту на ревью и **НЕ** морозит её (это метаданные
   процесса, не изменение контента). На ревью карта уходит только если на шаге 1 перенесли картинку.

Новые заметки **без** дубля (тир пуст / все кандидаты отвергнуты) — тоже пометить
`ai_generated::deduped` (проверены, дублей нет), без suspend и без `to_review`. Их список —
в конце `removal_candidates_review.md`.

### 5. Проверить и отчитаться

- `notesInfo`/`cardsInfo`: картинки перенесены (поле `Image` непусто, без остаточных проблем
  рендера), новые с картинкой — `suspend` + `ai_generated::to_review`, все обработанные новые —
  `ai_generated::deduped`, старые дубли — `to_remove`.
- Краткая сводка: сколько пар обработано, сколько картинок перенесено, сколько старых помечено
  `to_remove`, сколько новых `deduped`, что осталось на решение пользователя.
- Напомнить: **старые карты `to_remove` пользователь удаляет сам** в Anki, просмотрев
  `tag:to_remove`.

## Заметки

- Скан read-only и идемпотентный: повторный прогон пропускает уже `deduped` новые заметки, так что
  скилл можно гонять по мере наполнения новых колод.
- Перед вписыванием картинки всегда проверять, что `Image` новой карты пуст (не задваивать).
- Поле `Image` есть у `AiGeneratedJapaneseWords` (поля/шаблон — `knowledge/words-cards.md`). У типов
  `Kanji`/`Grammar` картинки обычно не нужны; если новая карта не «Слова» — перенос картинки, как
  правило, пропускается.
- `to_remove` уже использовался в прошлом проходе — тег тот же, ничего нового заводить не надо.
