# `AiGeneratedJapaneseWords` — карточки слов (лексика)

> **STATUS:** источник правды по типу «Слова» — **поля, правила их заполнения, теги уровня,
> HTML + JS + CSS**. Совпадает с тем, что реально стоит в Anki. Менять шаблон/стили — сначала тут,
> потом в Anki. Чем этот тип отличается от других (scope) — в [card-types.md](card-types.md) разд. 1;
> общие правила (безопасность, теги, suspend, фуригана, словарь, механика `anki_add.py`) —
> в [conventions.md](conventions.md).

Колода `新日本語::Слова`. Карточка слова: лицо JP→RU показывает слово с фуриганой, оборот — русское
определение, нюанс и примеры. При непустом `Bidi` генерируется и обратная карточка RU→JP.

## Поля и правила заполнения

`Kana` · `Kanji` · `Word` · `Definition` · `HintRU` · `HintJP` · `Examples` · `NoteRU` · `Image` · `Bidi` · `Remarks`

(`Kana` — первое поле и **sort field**: иначе кана-слова с пустым `Kanji` Anki отклоняет как
«пустые», а в браузере у них пустая колонка сортировки.)

1. **`Kana`** — чтение слова хираганой/катаканой (всегда заполнено). Сверять с `reading`/`readings`
   из `dict_lookup.py` — не выдумывать чтение кандзи.
2. **`Kanji`** — запись кандзи, если есть; для чисто-кана слов пусто. **Орфография по узусу, а не по
   вводу пользователя** (пользователь часто пишет наспех каной):
   - слово **широко пишут кандзи** (даже если дана кана) → **всегда** кандзи (`挨拶`, `鍵`, `机`,
     `椅子`, `携帯`);
   - кандзи **редкое / обычно пишут каной** → оставить каной (`たぶん` 多分 чаще кана, `きれい`,
     `ありがとう`); подсказка `uk: true` («usually kana alone») из словаря → оставить каной.
   - Спорные (≈50/50) решаю в пользу узуса и помечаю в черновике, чтобы пользователь переопределил.
3. **`Word`** — заголовок с фуриганой bracket-нотацией (рисуется на лице через `{{furigana:Word}}`).
   - С кандзи: сегментированное чтение, окуригана **вне** скобок: `切符[きっぷ]`, `食[た]べる`,
     `新[あたら]しい`.
   - Чисто-кана слово: просто кана **без скобок** (`たぶん`) — фильтр покажет её как текст.
4. **`Definition`** — краткое русское определение. Опираться на `meanings_ru` из словаря (если пусто —
   на `meanings_en`), а не на память; выбирать значение для N5–N4, при необходимости уточнять в
   скобках («билет (на транспорт)»).
5. **`HintRU`** — ограничитель синонимов на лице **обратной** (RU→JP) карточки. Заполнять, когда на
   русское определение претендует несколько японских слов: `(не チケット)`, `(не もしかして・きっと)`,
   коллокации (`歯を磨く`). Иначе пусто. Рендерится через `{{furigana:HintRU}}`, поэтому кандзи-
   конкуренты можно давать с фуриганой (`(не 眠[ねむ]る)` — пробел перед группой кандзи обязателен).
6. **`HintJP`** — уточнение на лице **прямой** (JP→RU) карточки для омографов / слов с несколькими
   значениями. Обычно пусто.
7. **`NoteRU`** — свободный нюанс на русском для понимания слова (рендерится серым италиком на
   обороте, **под словом, к которому относится**). Заполнять, только если есть действительно важный
   комментарий (напр. разница `手帳` vs `ノート`); иначе пусто. Это объяснение, в отличие от
   короткого `HintRU`.
8. **`Examples`** — 1–2 **коротких** предложения, где слово реально встречается (цель — удачный
   пример, а не объём). Каждый пример — две строки готовой HTML-разметки (фуригана только здесь):
   `<div class="exj">駅[えき]で 切符[きっぷ]を 買[か]いました。</div><div class="tr">Купил билет на станции.</div>`
   Несколько примеров — повторить пару `exj`+`tr`.
9. **`Image`** — обычно пусто. Заполнять только для конкретных существительных, где картинка помогает
   (содержимое — `<img src="файл">`, файл заранее положить через `storeMediaFile`).
10. **`Bidi`** — двусторонность. **Всегда непусто (двусторонняя)** для любой записи: пользователь сам
    уберёт обратную сторону, если не нужна.
11. **`Remarks`** — служебное поле правок; скилл создания его не заполняет (обрабатывает
    `process-remarks`); нигде не рендерится.

### Маппинг свободной инструкции пользователя на поля

Инструкция после `|` (синтаксис — в [conventions.md](conventions.md)) **приоритетнее** дефолтов и в
черновике должна быть видимо учтена:
- ограничение синонимов («не X», «≠ X», «только …») → `HintRU`;
- развёрнутое пояснение смысла («ежедневник с календарём, не блокнот») → `NoteRU`;
- «односторонняя»/«двусторонняя» → решение по `Bidi`.

### Маппинг полей `dict_lookup.py` (слова)

`{reading, readings, meanings_ru, meanings_en, pos, common, uk, jlpt, jlpt_tag, matched}`:
- **Kana** ← сверить с `reading`/`readings`;
- **Definition** ← `meanings_ru` (иначе `meanings_en`), значение для N5–N4;
- **Kanji** ← `uk: true` подсказывает оставить каной;
- `matched: "none"` — слова нет в словаре: повод насторожиться (опечатка / редкое / фраза);
  показать в черновике, не молчать.

## Теги уровня (JLPT / common) — только для этого типа

На `AiGeneratedJapaneseKanji` и `AiGeneratedJapaneseGrammar` эти теги **не ставятся** (там не лексика).

- Запрашивать словарь **по форме `Kanji`** (если есть), иначе по `Kana`.
- `common: true` → тег `common`; `jlpt_tag` (напр. `jlpt::n5`) — ставить, только если непустое.
  `jlpt::*` — **иерархический** тег по **самому лёгкому** уровню (максимальный номер N).
- **Фразы/приветствия — НИКОГДА не тегировать** (`こちらは〜さんです`, `お国はどちらですか` и т.п.),
  даже если словарь что-то вернул по слову внутри фразы. Косвенный признак фразы —
  `matched: "first"`/`"none"`, но решает смысл записи: одиночное слово в форме (`違います` → 違う)
  тегировать можно, устойчивую фразу — нет.
- В черновике-таблице показывать колонку `jlpt/common`, чтобы решение было видно.
- Источник — локальный словарь (`jlpt`/`common` из `dict_lookup.py`), см. [conventions.md](conventions.md).
  Теги — метаданные для фильтрации (custom study), содержимое карточки не меняют; JLPT-списки
  неофициальные — ориентир, не истина.

## Решения по шаблону

- Двусторонние карточки управляются полем `Bidi`: непусто → генерируется обратная (RU→JP).
- На лице JP→RU показываем **слово с фуриганой** (`Word`) — это карточки слов, не чтений.
- `Examples` — одно поле; несколько примеров кладём в него же; фуригана **только здесь**; фильтр
  `{{furigana:Examples}}` трогает только ` 漢字[かな]`, HTML-теги пропускает.
- `Image`: на JP→RU — **только на обороте** (на лице выдало бы ответ); на RU→JP — **на обеих
  сторонах** (картинка — ещё одно представление смысла-подсказки). На обороте RU→JP картинка стоит
  **между `Word` и примерами**, поэтому оборот RU→JP **не использует `{{FrontSide}}`** (лицо
  перерисовывается явно). Высота — `max-height: 250px` (в px, чтобы размер был одинаков на телефоне
  и компьютере).
- **Адаптивный размер заголовка `.word`:** JS shrink-to-fit (старт 40px, шагами −2px до 28px, иначе
  перенос) — чистым CSS нельзя, размер зависит от длины текста.

## Шаблоны

Карточка 1 (JP→RU, всегда):
```html
<!-- Front -->
<div class="word">{{furigana:Word}}</div>
{{#HintJP}}<div class="hint">{{HintJP}}</div>{{/HintJP}}
<!-- + скрипт автоподгона (см. ниже) -->
<!-- Back -->
{{FrontSide}}
<hr>
<div class="def">{{Definition}}</div>
{{#NoteRU}}<div class="note">{{furigana:NoteRU}}</div>{{/NoteRU}}
{{#Image}}<div class="img">{{Image}}</div>{{/Image}}
{{#Examples}}<div class="ex">{{furigana:Examples}}</div>{{/Examples}}
```

Карточка 2 (RU→JP, только если `Bidi` непусто):
```html
<!-- Front -->
{{#Bidi}}
<div class="def">{{Definition}}</div>
{{#HintRU}}<div class="hint">{{furigana:HintRU}}</div>{{/HintRU}}
{{#Image}}<div class="img">{{Image}}</div>{{/Image}}
{{/Bidi}}
<!-- Back (НЕ {{FrontSide}} — иначе Image окажется в середине, над ответом) -->
{{#Bidi}}
<div class="def">{{Definition}}</div>
{{#HintRU}}<div class="hint">{{furigana:HintRU}}</div>{{/HintRU}}
{{/Bidi}}
<hr>
<div class="word">{{furigana:Word}}</div>
{{#NoteRU}}<div class="note">{{furigana:NoteRU}}</div>{{/NoteRU}}
{{#Image}}<div class="img">{{Image}}</div>{{/Image}}
{{#Examples}}<div class="ex">{{furigana:Examples}}</div>{{/Examples}}
<!-- + скрипт автоподгона (см. ниже) -->
```

### Адаптивный размер заголовка (shrink-to-fit)

JS лежит в лице `JP->RU` (на оборот попадает через `{{FrontSide}}`) и в обороте `RU->JP`. Селектор
`.word` (а не id), чтобы не ловить дубли при `{{FrontSide}}`.

```html
<script>
(function(){
  function fit(){
    var els=document.querySelectorAll(".word");
    for(var i=0;i<els.length;i++){
      var el=els[i], min=28, size=40;
      el.style.whiteSpace="nowrap";
      el.style.fontSize=size+"px";
      while(size>min && el.scrollWidth>el.clientWidth){ size-=2; el.style.fontSize=size+"px"; }
      if(el.scrollWidth>el.clientWidth) el.style.whiteSpace="normal";
    }
  }
  if(typeof requestAnimationFrame!=="undefined") requestAnimationFrame(fit); else fit();
})();
</script>
```

## CSS (Styling)

Базовый шрифт ставит **кириллический санс-сериф первым**, японские шрифты — дальше (браузер берёт
глифы пофайлово); иначе русский текст падает в кривой моноширинный fallback. `.word` — размер ставит
JS (CSS задаёт только цвета/отступы). Примеры: `.exj` 24px, перевод `.tr` 18px серым.

```css
.card { font-family: -apple-system, "Helvetica Neue", Arial, "Hiragino Kaku Gothic ProN", "Yu Gothic", sans-serif; font-size: 22px; text-align: center; color: #222; background: #fff; }
.word { font-size: 40px; margin: 6px 0 10px; }
.def { font-size: 22px; margin: 8px 0; }
.hint { font-size: 18px; color: #c0392b; }
.note { font-size: 18px; color: #888; font-style: italic; margin-top: 10px; text-align: center; }
.ex { margin-top: 12px; text-align: left; display: inline-block; }
.exj { font-size: 24px; color: #333; margin-top: 10px; }
.tr { font-size: 18px; color: #888; margin-bottom: 2px; }
.img { margin: 10px 0; }
.img img { max-width: 90%; max-height: 250px; display: block; margin: 10px auto; }
hr { border: none; border-top: 1px solid #ccc; margin: 12px 0; }
ruby rt { font-size: 0.5em; color: #888; }
```

## Чек-лист на партию слов

- [ ] Поля заполнены по правилам выше; `Bidi` всегда непусто (двусторонняя).
- [ ] `Word` — bracket-форма (окуригана вне скобок); для кана-слов — кана без скобок.
- [ ] `Kana`/`Definition` сверены по `dict_lookup.py` (чтение/значение не выдуманы).
- [ ] Примеры короткие, с фуриганой и переводом (`.exj`+`.tr`), показывают употребление.
- [ ] Теги уровня проставлены по словарю (фразам — никогда); колонка `jlpt/common` видна в черновике.
- [ ] `canAddNotes` пройден; заметки в `新日本語::Слова` с `ai_generated` + `ai_generated::to_review`.
- [ ] Все карточки заморожены (`suspend`); двусторонние → 2 карты, проверено через `cardsInfo`.
