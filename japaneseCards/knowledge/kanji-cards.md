# `AiGeneratedJapaneseKanji` — карточки чтения слов в кандзи

> **STATUS:** источник правды по типу «Чтение кандзи» — **поля, правила их заполнения, HTML + CSS**.
> Совпадает с тем, что реально стоит в Anki. Менять шаблон/стили — сначала тут, потом в Anki. Чем
> этот тип отличается от других (scope) — в [card-types.md](card-types.md) разд. 2; общие правила
> (безопасность, теги, suspend, фуригана, словарь, механика `anki_add.py`) — в
> [conventions.md](conventions.md).

Колода `新日本語::Чтение кандзи`. Задача карточки: увидеть слово в кандзи → прочитать. Сюда же идёт
всё «на чтение»: числа, даты, счётчики, месяцы, дни недели, время. Любимый рабочий тип — просто
переносим слова в новый note type.

## Поля и правила заполнения

`Kanji` · `Kana` · `Meaning` · `Hint` · `Remarks`

1. **`Kanji`** — слово, записанное кандзи (можно и N4-слова с продвинутыми кандзи).
2. **`Kana`** — чтение (это ответ). Проверить по `dict_lookup.py --kanji …`, что чтение реально есть
   среди `on_yomi`/`kun_yomi` соответствующих кандзи. Кун пишется с точкой-разделителем окуриганы
   (`した.がう`) — при сверке точку игнорировать.
3. **`Meaning`** — русское значение, для справки/ассоциации; вспоминать не обязательно.
4. **`Hint`** — почти всегда **пусто**. Тип чтения (`訓読み`/`音読み` и т.п.) указывать **только когда
   это нужно** для снятия неоднозначности. Писать **с фуриганой в bracket-нотации, без квадратных
   скобок** вокруг: `訓[くん]読[よ]み`, `音[おん]読[よ]み`. Поле рендерится через `{{furigana:Hint}}`,
   поэтому фуригана работает в любой подсказке (обычный русский/японский текст без скобок проходит как есть).
5. **`Remarks`** — служебное поле правок; скилл создания его не заполняет (обрабатывает
   `process-remarks`); нигде не рендерится.

**Теги уровня JLPT/common НЕ ставить** (это чтение, не лексика). Только `ai_generated`
(+ `ai_generated::to_review`).

### Маппинг полей `dict_lookup.py --kanji`

`{on_yomi, kun_yomi, meanings_en, jlpt}` из KANJIDIC2 — проверить, что `Kana`-чтение карточки есть
среди `on_yomi`/`kun_yomi` соответствующих кандзи (кун — игнорируя точку окуриганы).

## Решения по шаблону

- `Kana` (ответ) рисуется **чёрным** (`#222`) как важное поле, не серым.
- `ruby rt` в CSS нужен, т.к. `Hint` рендерится через `{{furigana:Hint}}`.

## Шаблоны

```html
<!-- Front -->
<div class="word">{{Kanji}}</div>
{{#Hint}}<div class="hint">{{furigana:Hint}}</div>{{/Hint}}
<!-- Back -->
{{FrontSide}}
<hr>
<div class="kana">{{Kana}}</div>
<div class="def">{{Meaning}}</div>
```

## CSS (Styling)

Тот же кириллица-первой шрифт, что и у слов; `.kana` — ответ, **чёрный** (`#222`).

```css
.card { font-family: -apple-system, "Helvetica Neue", Arial, "Hiragino Kaku Gothic ProN", "Yu Gothic", sans-serif; font-size: 22px; text-align: center; color: #222; background: #fff; }
.word { font-size: 48px; margin: 10px 0; }
.kana { font-size: 24px; color: #222; }
.def { font-size: 22px; margin: 8px 0; }
.hint { font-size: 18px; color: #c0392b; }
hr { border: none; border-top: 1px solid #ccc; margin: 12px 0; }
ruby rt { font-size: 0.5em; color: #888; }
```

## Чек-лист на партию чтений

- [ ] `Kana`-чтение сверено по `dict_lookup.py --kanji` (есть среди on/kun соответствующих кандзи).
- [ ] `Hint` пуст, кроме случаев реальной неоднозначности чтения (тогда `訓[くん]読[よ]み` с фуриганой).
- [ ] Теги уровня НЕ проставлены; есть `ai_generated` + `ai_generated::to_review`.
- [ ] `canAddNotes` пройден; заметки в `新日本語::Чтение кандзи`.
- [ ] Карточки заморожены (`suspend`); проверено через `cardsInfo`.
